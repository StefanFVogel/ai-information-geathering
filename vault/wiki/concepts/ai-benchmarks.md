# AI Benchmarks

Standardisierte Tests zum Vergleich von AI-Modellen. Wichtig, aber häufig unter Vorbehalt zu lesen — Benchmark-Narrativ kann Marketing-getrieben sein.

## Wichtige Benchmark-Familien

- **SWE-Bench / SWE-Bench Pro** — Software-Engineering-Tasks aus GitHub-Issues
- **Vision / Document Reasoning** — multimodale Verständnistests
- **Rakuten Production-Task-Resolution** — Dritt-Benchmark mit realen Aufgaben
- **Bridge-Bench** — Halluzinations-Benchmark (viraler Quality-Indikator)
- **Misaligned Behavior Benchmark** — Safety-Metrik (niedriger = besser)

## Vorbehalte (Fall Graphify-71,5x)

Benchmark-Szenarien bilden Real-World-Nutzung oft nicht ab. Beispiel: [[Graphify]] wirbt mit 71,5x Token-Reduktion — verglichen mit einem Szenario ("alle 52 Files ins Context laden"), das niemand so nutzt. Real-World-Einsparung: ~7-8%.

## Vorbehalte (Fall Opus 4.6 → 4.7)

Jede [[Opus 4.6]]-Beschwerde findet im 4.7-Marketing einen passenden Benchmark-Sieg. Nate Herks Einordnung: "Als würden sie Löcher graben, um sie zu füllen." Empfehlung: Benchmarks mit eigenen Workflows gegentesten.
